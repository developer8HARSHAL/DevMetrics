## ğŸš€ Quick Start Guide

### For End Users (Integrate SDK)

#### **Step 1: Install the SDK**
```bash
npm install devmetrics-sdk
```

#### **Step 2: Get Your API Key**
Contact the DevMetrics administrator to get your unique API key:
```
Format: dm_[64-character-hex-string]
Example: dm_a1b2c3d4e5f6789...
```

#### **Step 3: Initialize in Your App**

**React / Vite:**
```javascript
// src/main.jsx or src/index.js
import { init } from 'devmetrics-sdk';

init({
  apiKey: import.meta.env.VITE_DEVMETRICS_KEY,
  backendUrl: 'https://devmetrics-backend.onrender.com',
  trackFetch: true  // Auto-track all fetch() calls
});
```

**Next.js:**
```javascript
// app/layout.js
'use client';
import { useEffect } from 'react';
import { init } from 'devmetrics-sdk';

export default function RootLayout({ children }) {
  useEffect(() => {
    init({
      apiKey: process.env.NEXT_PUBLIC_DEVMETRICS_KEY,
      backendUrl: 'https://devmetrics-backend.onrender.com',
      trackFetch: true
    });
  }, []);

  return <html lang="en"><body>{children}</body></html>;
}
```

**Node.js / Express:**
```javascript
import express from 'express';
import { init, track } from 'devmetrics-sdk';

const app = express();

// Initialize SDK
init({
  apiKey: process.env.DEVMETRICS_API_KEY,
  backendUrl: 'https://devmetrics-backend.onrender.com'
});

// Track all routes with middleware
app.use(async (req, res, next) => {
  const start = Date.now();
  
  res.on('finish', async () => {
    await track({
      endpoint: req.originalUrl,
      method: req.method,
      status: res.statusCode,
      responseTime: Date.now() - start
    });
  });
  
  next();
});

app.listen(3000);
```

#### **Step 4: View Your Dashboard**
Visit: **https://dev-metrics-six.vercel.app**

That's it! All your API calls are now being tracked. ğŸ‰

### SDK Configuration Options

| Option | Type | Required | Default | Description |
|--------|------|----------|---------|-------------|
| `apiKey` | `string` | âœ… Yes | - | Your DevMetrics API key |

| `trackAxios` | `boolean` | No | `false` | Auto-track axios requests |

### SDK API Reference

#### `init(options)`
Initialize the DevMetrics SDK.

```javascript
import { init } from 'devmetrics-sdk';

init({
  apiKey: 'dm_your_key_here',
  backendUrl: 'https://devmetrics-backend.onrender.com',
  trackFetch: true,
  trackAxios: false
});
```

**Returns**: Configuration object

---

#### `track(data)`
Manually track an API request or custom event.

```javascript
import { track } from 'devmetrics-sdk';

await track({
  endpoint: '/api/users',
  method: 'GET',
  status: 200,
  responseTime: 145,
  timestamp: new Date().toISOString() // Optional
});
```

**Parameters**:
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `endpoint` | `string` | âœ… Yes | API endpoint or URL |
| `method` | `string` | No | HTTP method (GET, POST, etc.) |
| `status` | `number` | âœ… Yes | HTTP status code (100-599) |
| `responseTime` | `number` | No | Duration in milliseconds |
| `timestamp` | `string` | No | ISO 8601 timestamp |

---

#### `getConfig()`
Get current SDK configuration.

```javascript
import { getConfig } from 'devmetrics-sdk';

const config = getConfig();
console.log(config);
// {

```

---

#### `wrapFetch()`
Manually enable fetch tracking (if not enabled during init).

```javascript

wrapFetch();
```

---

#### `wrapAxios()`
Manually enable axios tracking (if not enabled during init).

```javascript
import { wrapAxios } from 'devmetrics-sdk';

```

### 1. **Tracking Endpoint** (Used by SDK)

#### `POST /track`
Receives and stores tracking data from SDK.

**Request Headers**:
```
Content-Type: application/json
```

**Request Body**:
```json
{
  "apiKey": "dm_abc123...",
  "endpoint": "https://api.example.com/users",
  "method": "GET",
  "status": 200,
  "responseTime": 145,
  "timestamp": "2024-01-15T10:30:00.000Z"
}
```

**Success Response** (201):
```json
{
  "success": true,
  "message": "Request tracked successfully",
  "id": "65a4b2c8d9e1f2g3h4i5j6k7"
}
```

**Error Responses**:
- `400` - Validation failed
- `401` - Invalid/expired API key
- `429` - Rate limit exceeded
- `500` - Server error

| **Method** | HTTP method | String | `GET`, `POST`, `PUT`, `DELETE` |
| **Status** | HTTP status code | Number (100-599) | `200`, `404`, `500` |
| **Response Time** | Duration in ms | Number | `145`, `1250` |
| **Timestamp** | When request occurred | ISO 8601 | `2024-01-15T10:30:00.000Z` |
| **API Key** | Associated key | String | `dm_abc123...` |

### Calculated Analytics

| Metric | Formula | Description |
|--------|---------|-------------|
| **Success Rate** | `(2xx + 3xx) / Total Ã— 100` | Percentage of successful requests |
| **Error Rate** | `100 - Success Rate` | Percentage of failed requests |
| **Avg Response Time** | `Sum(times) / Count` | Mean response duration |
| **Requests/Hour** | `Count per hour` | Hourly traffic volume |

## ğŸ” Security & Best Practices

### API Key Security

**Format**: `dm_[64-character-hex-string]`
- Generated using `crypto.randomBytes(32)`
- 256-bit entropy
- Collision probability: ~1 in 10^77

**Storage**:
âœ… **DO**:
- Store in environment variables
- Use `.env` files (add to `.gitignore`)
- Rotate keys regularly (every 90 days)
- Use different keys for dev/staging/prod
- Revoke compromised keys immediately

âŒ **DON'T**:
- Hardcode keys in source code
- Commit keys to Git
- Share keys via email/chat
- Use same key across multiple apps
- Expose keys in client-side code (except SDK usage)

### Rate Limiting

**Default Limits**:
- **10,000** requests per hour per key
- **100,000** requests per day per key

**How it works**:
1. Backend counts requests per API key
2. Checks hourly and daily limits
3. Returns 429 if exceeded
4. Includes `resetAt` timestamp in response

**Rate Limit Response**:
```json
{
  "success": false,
  "message": "Rate limit exceeded",
  "limit": 10000,
  "resetAt": "2024-01-15T11:00:00.000Z"
}
```

**Customizing Limits**:
```bash
# Admin can update limits via API
curl -X PUT https://devmetrics-backend.onrender.com/apikey/dm_abc123... \
  -H "x-admin-key: YOUR_ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "rateLimit": {
      "requestsPerHour": 50000,
      "requestsPerDay": 500000
    }
  }'
```

### Data Privacy

**What DevMetrics tracks**:
âœ… Endpoint URLs
âœ… HTTP methods
âœ… Status codes
âœ… Response times
âœ… Timestamps

**What DevMetrics NEVER tracks**:
âŒ Request/response bodies
âŒ Headers (except method)
âŒ Query parameters with sensitive data
âŒ Authentication tokens
âŒ User personal information
âŒ IP addresses

### HTTPS in Production

**Backend (Render)**:
- âœ… Automatic HTTPS
- âœ… TLS 1.2+ enforced
- âœ… Certificate auto-renewal

**Dashboard (Vercel)**:
- âœ… Automatic HTTPS
